# SANED: National Social Protection Intelligence Platform
## The Comprehensive Master Blueprint

> **Abstract:** The SANED platform represents the Sultanate of Oman's definitive transition to a unified, intelligent, and zero-trust social protection ecosystem. By replacing fragmented legacy applications with a centralized CoreX GPU-accelerated engine, a Medallion architecture Data Lakehouse, and an Agentic AI overlay, SANED scales dynamically to serve 1.2M+ active citizens with a 99.9% uptime SLA.

### Session 2: Infrastructure & Security Foundation
_Audience: Network Engineers, SecOps, DevOps, IT Leads_

The SANED ecosystem is built on a highly fortified, multi-availability zone Virtual Private Cloud (VPC-SANED-PROD) emphasizing strict traffic segregation, deep packet inspection, and horizontal scaling.

**Core Topologies:**
1. **DMZ (Perimeter):** Cloudflare CDN / WAF feeds to FortiGate Gen-7 Firewalls.
2. **App Tier (Internal VPC):** Node.js/Next.js pods running on standard Kubernetes (K8s) Control Plane. Microservices managed by Istio Service Mesh.
3. **Mgmt Tier (SOC):** Splunk SIEM logger, Keycloak Identity Manager, Jump Hosts via Bastion.

### Session 3: Data Analytics & Lakehouse
_Audience: Data Architects, Data Scientists, BI Engineers_

Modernizing the Social Protection Fund requires moving beyond standard CRUD interfaces to predictive, deeply analytical data models.

**The Medallion Approach:**
- **Bronze (Landing Zone):** Raw JSON, internal records, external APIs from external ministries. (Immutable, Parquet format).
- **Silver (Standardized):** Quality-checked, normalized entity resolution. PII masked and Vault-encrypted.
- **Gold (Aggregated):** Aggregated data marts ready for analytical BI dashboards or directly triggering Machine Learning prediction models.

### Session 4: AI & Agentic MLOps
_Audience: Principal Machine Learning Engineers, AI Product Owners_

At the apex of the SANED intelligence layer sits the dynamic AI architecture, moving basic "if-then" workflows into intelligent, generative orchestration.

**Key Engines:**
- **MLOps Training Pipeline:** Continuous training using PyTorch distributed nodes processing Gold Zone data, managed by MLflow.
- **Vector Embeddings Vault:** Citizen interactions stored chronologically in Qdrant Vector databases, powering complex Retrieval-Augmented Generation (RAG).
- **Agentic Workers:** LLM-backed 'Agents' (using Llama-3/Claude via NVIDIA Triton) evaluating grant eligibility, automated document verification, and proactive service recommendations instantly.
